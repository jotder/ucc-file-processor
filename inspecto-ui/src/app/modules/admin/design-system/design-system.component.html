<inspecto-alert> for inline, per-screen notices (writes disabled, feature unavailable, jar missing) — instead of re-rolling bg-amber-100 … boxes. The message is projected, so it can carry <code> and links. Distinct from the always-mounted <inspecto-connectivity-banner>.

Schema-driven form

<inspecto-schema-form> renders a Component Type's AttributeSpec[] with the platform's three-tier disclosure: required fields up front, optional collapsed, advanced behind the gear. dependsOn attributes show (and validate) only while their controller matches — switch the protocol to see the Host field appear and disappear. Prefer this over a hand-built form whenever the fields are plain config attributes. 

{{ snippets.schemaForm }}

Data table

One <inspecto-data-table [tier]> (inspecto/data-table) consolidates every grid. Tiers, like phone versions: mini = the grid; standard = + search · filter · sort · CSV export; pro = + the offline SQL/query editor (projection, nested AND/OR, Advanced SQL); pro max = + save the query as a reusable rule template. The heavy logic lives in framework-free core/ (csv · quick-filter · controller) + inspecto/query; the component is a thin shell.

Tree table (aligned hierarchy)

<inspecto-tree-table> — an expandable indented hierarchy on the left with aligned value columns on the right (Community ag-Grid; no Enterprise tree-data). One component for breakdown-by-dimensions, multi-entry comparison, N-way reconciliation and file browsing. Cell plugins are ordinary columns ColDefs (here the Δ column uses varianceCell()); [rowActions] adds action icons and [multiSelect] adds checkbox selection. See tree-table-design.md.

Graph hosts (AntV G6)

Two patterns for node-graph views — both reuse canvasTheme() + nodeColor/nodeShape (never hardcode canvas colours). See the angular-ui skill §4. Live examples are on the Pipelines pane (Per-pipeline / Combined / Editor toggle) and the Link Analysis studio.

Read-only

catalog/graph-view.component — rebuilds the graph on every data / scheme change. For static visualisations (catalog graph, Pipelines Per-pipeline / Combined). Opt-in inputs (all default off; hosts stay byte-identical without them): [fill] grow-to-container, [display] per-kind colour/shape/pattern/size overrides, [tooltips] hover details, [emphasis] highlight/dim + community grouping, and [layout] — 11 selectable G6 layouts (GRAPH_LAYOUTS/layoutConfig(); the 3 tree layouts gate on the pure isForest()). Live with all options: the Link Analysis studio, which pairs it with the pure algorithm library inspecto/graph/graph-analysis.ts (paths, centrality, LP + Louvain communities, motif pattern matching).

Interactive editing (T32)

pipelines/pipeline-editor-graph.component — a persistent graph mutated in place (drop-to-add, two-click connect, delete), rebuilt only on subject change (graphKey). Live in the Pipelines Editor mode.

Map host (MapLibre GL)

inspecto/geo/map-view.component — the Geo Map Analysis analog of the read-only graph host: fully offline basemap (slimmed Natural Earth GeoJSON + bundled glyphs under assets/basemap/, no tile server), scheme-aware via theme/map-tokens.ts (the map's chart-tokens), pan/zoom, scale bar, fit-to-data, PNG export. Data plane: GeoData points render as circles; (pointClick) emits the point id. A pmtiles:// protocol is pre-registered for customer vector-tile archives (Phase 4).
